Conformance: rendering/multiple-render-targets

Hand-rolled to pin the WebGL 2 MRT surface this milestone exercises. Covers:

  • WebGL2RenderingContext available + acquired via canvas.getContext('webgl2')
  • MAX_DRAW_BUFFERS ≥ 2 and MAX_COLOR_ATTACHMENTS ≥ 2
  • COLOR_ATTACHMENT0 + COLOR_ATTACHMENT1 constants exposed
  • framebufferTexture2D attaching two color textures to one FBO at attachments 0 and 1
  • checkFramebufferStatus returns FRAMEBUFFER_COMPLETE for the 2-attachment FBO
  • drawBuffers accepts both a plain JS Array (Three.js's call shape) and a Uint32Array
  • GLSL ES 3.00 fragment shader with layout(location=0/1) out vec4 writing two distinct outputs in one draw
  • readBuffer + readPixels verifies each attachment received the right output
  • drawBuffers([NONE, COLOR_ATTACHMENT1]) masks the first attachment — subsequent readback from attachment 0 should show the prior clear color, not the new draw color
