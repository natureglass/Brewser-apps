Three.js r162 — Shader (Monjori)

Port of webgl_shader.html: a fullscreen PlaneGeometry(2, 2) rendered by an OrthographicCamera(-1, 1, 1, -1, 0, 1), drawn with a custom ShaderMaterial running the canonical "Monjori" (by Mic) shadertoy-style fragment shader — ~40 sin/cos/pow/sqrt/mod calls per fragment driven by a single time uniform that updates from performance.now() / 1000 each frame. Vertex shader is a pure passthrough (gl_Position = vec4(position, 1.0)), fragment shader is the procedural color pattern. Both shader source strings get #pragma raw_passthrough prepended so the bridge routes through the raw-shader passthrough path (memory), running the user's GLSL verbatim on native GLES.


No new bridge surface forced — the path was fully built by milestones #14 (custom-fragment-shader rendering via raw_passthrough), #20 (auto-promote markers in compileShader's source-scan), and #22 (custom-uniform-name forwarding via passthrough). This demo simply rides them: one custom uniform float time per frame, OrthographicCamera projection ignored by the shader (NDC fullscreen quad), no textures. Fragment shader complexity is the only thing to watch — ~40 trig + transcendental ops per fragment with no LODs, so 640×360 should be ~30+ FPS but 1280×720 fullscreen may dip to single digits. lil-gui not present in upstream; no DOM stubs needed.
