Three.js r184 — Sky + Sun Shader

1:1 port of webgl_shaders_sky.html from three.js latest (r184). Skydome based on the Preetham analytical daylight model + a custom multi-octave cloud noise overlay; sun/cloud parameters editable via an UNPATCHED upstream lil-gui panel mounted into the live-DOM stack (M2.0–M2.5). This page exists to stress-test the Phase 2 DOM surface against a real-world lil-gui consumer — expect rough edges; we screenshot what shows and triage from there.


No gamepad input — the camera is fixed at (0, 100, 2000). The sky animates via the time uniform driving the cloud noise.

Pre-authorized deviations from upstream: (1) Three.js r184 loaded via the __THREE_R184_STAGED__ IIFE + the three nx.js shims at libs/r184-nxjs-bridge.js. (2) lil-gui loaded as libs/lil-gui-unpatched.iife.js — byte-identical to upstream lil-gui.module.min.js modulo IIFE wrap + globalThis.lilGui assignment. (3) Sky.js ES module → IIFE; globalThis.Sky. (4) OrbitControls dropped entirely — static camera. (5) Canvas reused from <canvas id="sky-canvas">; no document.body.appendChild(renderer.domElement) / setPixelRatio / resize listener. (6) WebGL 2 context with the constructor.name = 'WebGL2RenderingContext' Proxy + bypassWebGL1Check. (7) console.warn/log/error/info silenced. (8) renderer.resetState() before every render.
