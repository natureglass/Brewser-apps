Three.js r184 — WebGL 2 Volume Render (3D Texture)

Port of webgl_texture3d.html from Three.js r184 (first r184 demo in this suite). A Data3DTexture with RedFormat + FloatType (R32F) holds the volumetric data from stent.nrrd (881 KB, medical imaging "stent" scan). A ShaderMaterial raymarches through the box mesh via sampler3D, mapping density to color via a 1D colormap texture. Left stick orbits the camera. A cycles render style (MIP ↔ ISO); X cycles colormap (viridis ↔ gray).


Conformance: textures/webgl2-texture3d — pins the 3D texture surface this milestone exercises: bindTexture(TEXTURE_3D), texImage3D with RGBA8 and R32F (RedFormat + FloatType) format/type combos, texParameteri(TEXTURE_3D, ...), full passthrough draw sampling a 2×2×2 R32F 3D texture via sampler3D + texture(u_data, vec3 uv) + readPixels verifying the sampled values.

First r184 demo in this suite — loads via libs/three-latest.iife.js instead of the r162 IIFE. NRRDLoader + VolumeRenderShader1 bundled as libs/nrrd-loader.iife.js (~653 KB; includes its own copy of the THREE classes it needs internally). The shader uses precision mediump sampler3D; uniform sampler3D u_data; — GLES 3.00 syntax. Runs natively on Tegra GLES3 via the raw-shader passthrough path (Three.js auto-emits #define SHADER_NAME for ShaderMaterial which triggers the auto-promote). The volume ranges from 0 (transparent) to 1 (opaque); MIP mode shows the maximum intensity along each ray, ISO mode shows the first surface where density crosses the threshold.
