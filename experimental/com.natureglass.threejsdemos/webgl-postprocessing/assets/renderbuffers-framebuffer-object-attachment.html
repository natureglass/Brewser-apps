FBO conformance: renderbuffers/framebuffer-object-attachment

Hand-rolled to pin the FBO management surface added in milestone #19 (bridge-fbo-support). Adapted from the Khronos conformance test of the same name, scoped to the API shape and error returns nx.js actually implements (we don't surface DEPTH_STENCIL_ATTACHMENT, INVALID_FRAMEBUFFER_OPERATION, or the RED_BITS/DEPTH_BITS/etc. getParameter pnames upstream relies on for buffer-bits introspection). Covers:

  • createFramebuffer / deleteFramebuffer / isFramebuffer round-trip
  • bindFramebuffer with valid FBO + null + deleted
  • createRenderbuffer / deleteRenderbuffer / isRenderbuffer round-trip
  • renderbufferStorage with DEPTH_COMPONENT16 + invalid pname rejection
  • framebufferTexture2D with COLOR_ATTACHMENT0
  • framebufferRenderbuffer with DEPTH_ATTACHMENT
  • checkFramebufferStatus returns FRAMEBUFFER_COMPLETE for a properly-built FBO
  • Render-into-FBO + readback verifies pixel output reaches the FBO's color attachment texture
