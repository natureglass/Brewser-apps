Three.js r162 — Skinning + Animation Blending (PARKED — Mesa Nouveau)

Milestone #26 parked on Citron, 2026-05-22. Port of webgl_animation_skinning_blending.html: Mixamo Vanguard Soldier (49 bones, 4 clips) blended via AnimationMixer. The WebGL 2 surface itself is fully functional — screen.getContext('webgl2') returns a real WebGL2RenderingContext, GLSL ES 3.00 shaders compile and link, all WebGL 2 entry points work (memory:nxjs-webgl2-surface). The skinning specifically hits a Mesa Nouveau driver bug: Three.js's skinning_pars_vertex uses texelFetch(boneTexture, ivec2(x, y)) with runtime-computed coordinates derived from skinIndex.x. On Citron's Mesa Nouveau, vertex-shader sampling of RGBA32F immutable storage returns garbage when coordinates are computed at runtime; static coords work fine. We verified per-vertex skinIndex.x reaches the shader correctly, multi-coordinate static texelFetch returns real bone matrix data, and the bone texture is uploaded correctly — the bug is in the driver's handling of dynamic texel coords. Unrolling into a switch with constant coords per case didn't help. May render correctly on real Tegra hardware; deferred until hardware verification is possible.

R = crossfade run → walk (5 s). Camera orbits with the left analog stick via SwitchOrbitControls ([[swb-threejs-misc-controls-orbit]]).
