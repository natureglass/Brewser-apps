Khronos conformance: rendering/gl-drawelements

Ported from WebGL/sdk/tests/conformance/rendering/gl-drawelements.html. Validates gl.drawElements: legal primitive modes (TRIANGLES) succeed, desktop-GL-only modes (QUADS, QUAD_STRIP, POLYGON) return INVALID_ENUM; legal index types (UNSIGNED_BYTE, UNSIGNED_SHORT) succeed, illegal types (UNSIGNED_INT in WebGL 1, FLOAT, SHORT) return INVALID_ENUM; and basic index bounds-checking returns INVALID_OPERATION for out-of-range indices. Milestone #8 (webgl-sprites) surface: Three.js's Sprite uses an indexed quad geometry, so every sprite draw flows through gl.drawElements + the bridge sprite-draw path. This test pins the JS-side validation gates that sit in front of the bridge dispatch.
