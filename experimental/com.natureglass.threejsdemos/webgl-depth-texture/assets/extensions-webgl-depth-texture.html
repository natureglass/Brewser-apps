Conformance: extensions/webgl-depth-texture

Hand-rolled to pin the WEBGL_depth_texture surface added in milestone #19.5 (bridge-depth-texture-support). Adapted from the Khronos conformance test of the same name, scoped to the API shape and error returns nx.js actually implements (we don't surface DEPTH_BITS/STENCIL_BITS getParameter pnames, INVALID_FRAMEBUFFER_OPERATION, copyTexImage2D, or copyTexSubImage2D that the upstream test depends on for its pixel-output verification path). Covers:

  • WEBGL_depth_texture listed in getSupportedExtensions + OES_depth_texture sibling listed too
  • getExtension('WEBGL_depth_texture') returns a non-null object exposing UNSIGNED_INT_24_8_WEBGL = 0x84FA
  • getExtension('OES_depth_texture') alias works the same
  • gl.UNSIGNED_INT_24_8_WEBGL constant defined on the context proto (0x84FA)
  • gl.DEPTH_COMPONENT constant defined (0x1902)
  • texImage2D happy-path for (DEPTH_COMPONENT, UNSIGNED_SHORT, null)
  • texImage2D happy-path for (DEPTH_COMPONENT, UNSIGNED_INT, null)
  • texImage2D happy-path for (DEPTH_STENCIL, UNSIGNED_INT_24_8_WEBGL, null)
  • texImage2D rejects non-null data on depth formats with INVALID_OPERATION
  • framebufferTexture2D with depth texture attachment succeeds
  • checkFramebufferStatus returns FRAMEBUFFER_COMPLETE for color + depth-texture attachments
  • Render-to-FBO smoke test: clear with depth, read back color attachment via readPixels (verifies the depth-texture-attached FBO is fully writable)
