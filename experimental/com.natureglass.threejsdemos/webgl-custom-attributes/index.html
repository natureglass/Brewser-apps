Three.js r162 — Custom Attributes

Port of webgl_custom_attributes.html: a high-res SphereGeometry(50, 128, 64) drawn with a ShaderMaterial whose custom vertex shader displaces each vertex along its normal by amplitude * displacement (a per-vertex custom-named float attribute rewritten via bufferSubData every frame), and whose fragment shader samples a tiled grayscale water.jpg texture and modulates by a hue-rotating color uniform. First milestone to exercise the bridge's passthrough sampling of CPU-data textures — a lazy-promote in try_draw_passthrough allocates a persistent GLES handle for the bound 2D texture on first passthrough use, mirroring the texel data via nx_webgl_egl_persistent_texture_image_2d. Also pins custom-attribute-name handling (attribute float displacement) and custom-uniform-name uploads (amplitude, colorTexture) through the raw-shader passthrough path (memory).


Conformance: passthrough/cpu-data-texture-sample — hand-rolled, pins the lazy-promote: creates a 4×1 RGBA UByte DataTexture, samples it from a #pragma raw_passthrough shader with a custom-named UV attribute, and asserts the readback pixels match the input. Also pins custom-attribute-location resolution (getAttribLocation on a non-allowlisted name returns a valid index) and custom-uniform forwarding (uniform1f/uniform3f/uniform1i on non-allowlisted names reach native GLES).

Texture is the same water.jpg as milestone #9 (webgl-geometry-dynamic); sourced from sdmc:/.../webgl-geometry-dynamic/assets/water.jpg per memory (Image bypass of brewser:// scheme). The displacement attribute is a 1-component float rewritten every frame via bufferSubData; sphere has 16,641 vertices, ~32,768 triangles, all flowing through native GLES via #pragma raw_passthrough injected into both shader source strings before ShaderMaterial sees them. lil-gui toggle dropped (single bool, no obvious gamepad mapping).
