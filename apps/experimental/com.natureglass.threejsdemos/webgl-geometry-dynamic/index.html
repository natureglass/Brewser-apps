Three.js r162 — Dynamic Geometry

Port of three-r162/examples/webgl_geometry_dynamic.html: a 128×128 vertex PlaneGeometry(20000, 20000, 127, 127) rotated horizontal, with every vertex's Y mutated each frame to make a rolling water surface. Mapped with a tiled grayscale water.jpg, tinted by MeshBasicMaterial({ color: 0x0044ff }), and faded by FogExp2(0xaaccff, 0.0007). First milestone to exercise the bridge's per-frame bufferSubData upload path at scale (~16k vertices/frame in the position attribute) — historically the exact surface that the now-fixed GpuCompositor bug used to corrupt.


Conformance: buffers/buffer-data-and-buffer-sub-data — pins the bufferData/bufferSubData validation surface this demo hammers (~16k vertices/frame).

Right stick looks around (upstream: mouse pointer position); B button moves forward (upstream: left click); A button moves backward (upstream: right click). Left stick also drives strafe + forward/back if you prefer analog control. The original demo's FirstPersonControls is replaced with a gamepad-driven sibling at libs/first-person-controls.js mirroring the same API surface.
