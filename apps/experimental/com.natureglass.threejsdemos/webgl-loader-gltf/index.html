Three.js r162 — GLTFLoader (model browser)

Adapted port of webgl_loader_gltf.html from three.js latest (r184) — the model browser, gamepad cycling, and HDR environment work. The r184 1:1 attempt landed on a Mesa Nouveau driver wedge in PMREM's blur ping-pong, so this build runs on r162 (smaller PMREM cubeUV layout + the WebGL 1 bridge path) until real Tegra hardware is available. Loads the Royal Esplanade RGBE equirect (royal_esplanade_1k.hdr) and prefilters it via PMREMGenerator for IBL; the helmet's visor reads as glass again. Cycles through four bundled glTF models: DamagedHelmet (PBR canonical), Duck (classic textured sample), Flamingo, and Horse. MeshStandardMaterial runs through Three.js's auto-generated GLSL via the bridge's SHADER_NAME passthrough.


Gamepad left stick orbits the camera; triggers zoom. A = previous model, B = next model. Each model swap re-runs fitCameraToSelection() and re-initialises any AnimationMixer clips upstream's demo would have played.

Pre-authorized deviations from upstream: (1) Three.js r162 instead of r184 (the upstream version) — r184's PMREMGenerator blur ping-pong hangs Citron's Mesa Nouveau backend, see memory; expected to clear on real Tegra hw. (2) UltraHDRLoader (r184-only) replaced with r162's RGBELoader + royal_esplanade_1k.hdr (asset already on disk; UltraHDRLoader port preserved at libs/ultrahdr-loader.js for the eventual re-attempt). (3) fetch(model-index.json) from GitHub replaced with a bundled hardcoded list of four pre-extracted models (Citron has no TCP sockets). (4) lil-gui dropdown replaced with gamepad A/B cycling. (5) scene.backgroundBlurriness = 0.0001 — routes WebGLBackground through the PMREM cubeUV path instead of the unsupported real WebGLCubeRenderTarget path. (6) OrbitControls mouse mapped to gamepad sticks. (7) renderer.compileAsync dropped — nx.js has no parallel-shader-compile. (8) Morph-target animations (Flamingo, Horse) do not play — our in-house GLTFLoader does not yet implement the weights animation path. Skeletal + TRS animations do play. Static meshes render correctly.
