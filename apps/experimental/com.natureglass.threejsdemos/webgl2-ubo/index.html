Three.js r184 — WebGL 2 Uniform Buffer Objects

1:1 port of webgl_ubo.html from three.js latest (r184): 200 alternating tetrahedrons + textured cubes, each with its own RawShaderMaterial({glslVersion:GLSL3}) clone sharing two UniformsGroup buffers (ViewData: projection + view matrices; LightingData: Phong parameters). All 200 materials write through ONE UBO per group via gl.bindBufferBase — uploaded once per frame, sampled by every program.


Conformance: buffers/uniform-buffer-objects — hand-rolled, pins the bridge surface this milestone widens: bindBuffer/bufferData/bufferSubData accepting all 6 WebGL 2 buffer targets, full UBO round-trip via bindBufferBase + uniformBlockBinding, plus an end-to-end passthrough draw that samples a UBO and verifies via readPixels that the value reached the shader.

Three.js's WebGLUniformsGroups binds each UniformsGroup via gl.bindBuffer(UNIFORM_BUFFER, buffer), gl.bufferData(UNIFORM_BUFFER, size, DYNAMIC_DRAW), gl.bindBufferBase(UNIFORM_BUFFER, slot, buffer), then per-frame writes via gl.bufferSubData(UNIFORM_BUFFER, offset, data). This milestone widens the bridge's buffer-target acceptance to the full WebGL 2 set and threads UBO indexed bindings through the passthrough dispatch path.
