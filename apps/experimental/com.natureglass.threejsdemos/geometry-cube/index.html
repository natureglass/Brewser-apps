Three.js r162 — Geometry Cube

Port of three-r162/examples/webgl_geometry_cube.html: a 1×1×1 BoxGeometry rotated each frame and textured with the crate image loaded from assets/crate.png (converted from the upstream textures/crate.gif since nx.js's image decoders only handle PNG/JPEG/WebP).


Conformance: rendering/gl-drawelements — pins the WebGL 1 validation gates in front of gl.drawElements (primitive-mode allowlist, index-type allowlist, index-bounds + non-aligned-offset rejection). Co-located with this demo because the textured-cube draw path is the canonical indexed-triangle dispatch through the bridge.

If the cube canvas is RED with "PRE-WEBGL" text, the external scripts didn't run - check the status canvas above for which step failed. Known limitation: one face of the cube renders with a brightness artifact on Switch hardware - see threejs-cube-white-face memory.
