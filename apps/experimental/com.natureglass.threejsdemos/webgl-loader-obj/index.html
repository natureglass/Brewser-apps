Three.js r162 — OBJ Loader

Port of webgl_loader_obj.html: the male02 OBJ model (~5000 triangles across 10 sub-meshes / 14 mesh nodes) loaded with OBJLoader. Upstream uses a single uv_grid_opengl.jpg UV-debug texture as a blanket override; this port deviates and instead loads the OBJ's own male02.mtl to apply each submesh's real source JPEG by material name (minimal inline MTL parser; MTLLoader's TextureLoader dep wasn't worth porting). Lit by an AmbientLight plus a camera-attached PointLight. First milestone to exercise an external-asset text loader, and the bridge's new derivative-normals fallback for flatShading: true submeshes (OBJLoader emits Phong materials with flatShading driven by the file's smoothing-group flags).


Conformance: extensions/oes-standard-derivatives — port of the Khronos test that verifies the extension is listed by getSupportedExtensions, that getExtension returns an object exposing FRAGMENT_SHADER_DERIVATIVE_HINT_OES (0x8B8B), that gl.hint(target, mode) accepts the new pname and rejects invalid modes, and that a passthrough fragment shader using dFdx / dFdy produces non-uniform pixel output.

Left stick orbits the camera (minDistance=2, maxDistance=5; pan + zoom disabled). Render-on-demand: only redraws when the orbit stick moves or on first frame after the texture + OBJ both finish loading. The bridge's derivative-normals fallback kicks in for any submesh where the OBJ's smoothing-off state put flatShading: true on the auto-generated MeshPhongMaterial — without it those parts of the figure would render unlit (the flatShading optimizer drops the normal attribute, the bridge's normals path then silently falls through to unlit; see bridge-flatshading-gap memory). Watch the status canvas's "extensions" row — the new OES_standard_derivatives entry alongside ANGLE_instanced_arrays is the milestone's outward sign.
