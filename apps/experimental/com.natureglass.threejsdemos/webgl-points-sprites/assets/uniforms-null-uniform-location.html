Khronos conformance: uniforms/null-uniform-location

Ported from WebGL/sdk/tests/conformance/uniforms/null-uniform-location.html. Validates that passing a null location to any uniform setter (uniform1f / uniform1fv / uniform1i / ... / uniformMatrix4fv) is a silent no-op generating NO_ERROR. Milestone #10 surface: Three.js materials upload uniforms by name, and when a name isn't in the active program (e.g. size on a non-PointsMaterial program, diffuse in a custom shader), getUniformLocation returns null and Three.js fires the setter anyway. The bridge must accept null cleanly so material code doesn't crash on programs that don't declare the uniform.
