Three.js r162 — Points Sprites

Port of three-r162/examples/webgl_points_sprites.html: a 10,000-point cloud (single shared BufferGeometry) reused across 5 THREE.Points instances, each with its own PointsMaterial({size, map: snowflakeN.png, blending: AdditiveBlending, transparent: true, depthTest: false}) at sizes 20/15/10/8/5 px. Per-frame HSL color animation per material, independent rotation per Points instance, FogExp2(0x000000, 0.0008). First milestone to exercise textured + sized + attenuated point sprites through the bridge (milestone #6 added the GL_POINTS path; this milestone extends it to dispatch via bridge_texture_program when a texture is bound, honor material.size for quad extent, and honor sizeAttenuation via Three.js's scale uniform).


Conformance: uniforms/null-uniform-location — pins the null-location no-op contract for every uniform setter. Three.js fires setters even for uniforms the active program doesn't declare (e.g. size on a non-PointsMaterial program).

10,000 points distributed in a 2000×2000×2000 cube, rendered 5 times with different sprite textures and sizes (20/15/10/8/5 px) + AdditiveBlending. Each Points instance has independent rotation; each material's HSL hue cycles per frame. Right stick (or left stick) pans the camera — upstream uses mouse position to drive camera.position.{x,y} with smoothing.
