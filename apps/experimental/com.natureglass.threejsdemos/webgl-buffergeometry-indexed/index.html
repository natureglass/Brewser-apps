Three.js r162 — BufferGeometry Indexed

Port of webgl_buffergeometry_indexed.html: a 20×20 grid built as an indexed BufferGeometry (11×11 = 121 vertices, 200 triangles, 600 indices) with per-vertex position, normal, and color attributes. Rendered with MeshPhongMaterial({ side: DoubleSide, vertexColors: true }) under a single HemisphereLight at intensity = 3. First milestone to exercise the bridge's new hemisphereLights[0] path — bridge_color_program now recognizes .direction / .skyColor / .groundColor uniforms and composes the irradiance mix(ground, sky, 0.5·dot(N, dir) + 0.5) into the ambient term, matching Three.js's lights_fragment_begin.glsl.js chunk.


No Khronos conformance test fits HemisphereLight directly (it's a Three.js abstraction, not a WebGL primitive). Visual verification is via the rotating grid above, with the status canvas confirming the active-uniform list contains the three hemisphereLights[0].* names. With useLegacyLights = false (the r162 default) and intensity = 3, Three.js uploads skyColor ≈ (3, 3, 3) and groundColor ≈ (0.8, 0.8, 0.8) — the bridge has no tone mapping so faces aligned with the up vector saturate. Per side: DoubleSide, the back face takes the ground-color-tinted irradiance because the bridge does not yet flip the geometric normal via gl_FrontFacing; visible as the grid rotates past 90°.
