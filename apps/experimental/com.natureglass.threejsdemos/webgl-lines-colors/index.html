Three.js r162 — Lines Colors

Port of three-r162/examples/webgl_lines_colors.html: six rotating THREE.Line objects, each carrying its own per-vertex color attribute. Material is shared LineBasicMaterial({ color: 0xffffff, vertexColors: true }) — the bridge's line program multiplies each fragment by v_color. Tilt the left analog stick to pan the camera (replacement for upstream's mouse-follow).


Conformance: attribs/gl-bindAttribLocation-aliasing — pins bindAttribLocation aliasing detection at link time, which the per-vertex-color demo relies on for binding a_color to its own slot.

Six rotating Hilbert-curve lines arranged in a 2×3 grid. The top row uses the raw 64 Hilbert points (jagged); the bottom row uses a Catmull-Rom spline through them × 6 subdivisions (smooth). Each line has independent per-vertex HSL coloring. Colors render slightly darker than desktop Three.js because the bridge's fixed-function output doesn't apply the final sRGB gamma curve.
