WebGL Experiments

Khronos conformance suites and Three.js r162 demos running inside an inline <canvas> via the GPU bridge. ThreeJS demos are split by which WebGL context they acquire: WebGL 2 demos call getContext('webgl2') and depend on the WebGL 2 surface; the rest ride the WebGL 1 bridge + raw-shader passthrough.

WebGL 2 ThreeJS Demos

Production demos exercising canvas.getContext('webgl2') + GLSL ES 3.00 shaders + WebGL 2-specific surface (MRT, UBOs, etc.). See nxjs-webgl2-surface.

  • webgl-texture3d
    #34 (🔒 PARKED on Mesa Nouveau): 3D volume renderer (stent.nrrd). Bridge surface complete (texParameteri target widening, TEXTURE_BINDING_3D, texImage3D binding slot); sampler3D returns vec4(0) on driver for ALL formats. r184 fetch-hang blocker also discovered.
  • webgl-animation-skinning-blending
    #26 (🔒 PARKED on Mesa Nouveau): WebGL 2 surface now wired but Citron's vertex texelFetch on RGBA32F returns garbage with runtime coords. Awaits real-Tegra hw verification.
  • webgl2-ubo
    #33 (🔒 PARKED on Mesa Nouveau): 80 tetra/box meshes sharing two UniformsGroup UBOs. Bridge widening shipped (6 v2 buffer targets + UBO re-apply); driver zeros 2nd UBO sample. Conformance 59/62. Expected to work on real Tegra hw.
  • webgl2-multiple-rendertargets
    #32: TorusKnot G-Buffer via RawShaderMaterial({glslVersion:GLSL3}) writing 2 outputs in one pass to a WebGLRenderTarget({count:2}); composite samples both. First production demo of the WebGL 2 surface.
  • webgl-texture2darray
    H3 (r184): 1:1 port of webgl_texture2darray.html. 256×256×109 RED volume uploaded as DataArrayTexture (TEXTURE_2D_ARRAY); custom GLSL3 ShaderMaterial samples slices via sampler2DArray with an animated depth uniform. Validates fragment-side sampler2DArray on Citron (separate from #34's vertex-texelFetch wedge).
  • webgl-shaders-sky
    H4 (r184): 1:1 port of webgl_shaders_sky.html. Preetham analytical skydome + cloud noise overlay (Sky.js IIFE), parameters editable via an UNPATCHED upstream lil-gui panel (lil-gui-unpatched.iife.js). Stress-tests the Phase 2 live-DOM stack (M2.0–M2.5) against a real-world consumer; OrbitControls dropped (static camera).
