Three.js r162 — WebGL Layers

Port of three-r162/examples/webgl_layers.html: 300 randomly-positioned BoxGeometry meshes split across three camera layers (red/green/blue), lit by a single PointLight attached to the orbiting camera. First milestone to exercise the bridge's point-light path (a_viewPosition + u_pointLightPosition/u_pointLightColor in bridge_color_program) with per-fragment light direction calc.

Toggle layers with: B red · A green · Y blue · X enable all (lil-gui replacement).


Conformance: programs/program-infolog — verifies getProgramInfoLog doesn't return a literal "\0". Layered scenes link multiple programs per render so correct info-log reporting matters when something goes wrong.

300 cubes scattered randomly in a ±20 world cube, split evenly across three layers (red / green / blue). The camera orbits at radius 5 around origin — so the camera passes through the scattered cubes. Press B/A/Y to toggle each color layer's visibility; press X to restore all three. The current camera layer mask is shown in the diagnostic above (R/G/B columns).
