Three.js r162 — Depth Texture

Port of webgl_depth_texture.html: 50 blue TorusKnotGeometry meshes randomly distributed on a sphere surface, rendered first into a WebGLRenderTarget with a sampleable depth texture attachment, then visualized by a fullscreen-quad post-pass that samples target.depthTexture and outputs 1.0 - depth as grayscale (closer = brighter). First milestone to exercise nx.js's new WEBGL_depth_texture extension exposure and depth-format texImage2D acceptance. Left stick orbits the camera (with damping). A cycles depth format (Depth ↔ DepthStencil); B cycles depth type (UnsignedShort ↔ UnsignedInt) when format is Depth. Drops in for upstream's lil-gui without a DOM dependency.


Conformance: extensions/webgl-depth-texture — hand-rolled to pin the surface added in this milestone: extension presence + UNSIGNED_INT_24_8_WEBGL constant, texImage2D with each depth/depth-stencil format/type combo, framebufferTexture2D with depth attachment, checkFramebufferStatus COMPLETE on a color + depth-texture FBO, and detach + re-attach round-trips.

Left stick orbits the camera. A cycles depth format (Depth ↔ DepthStencil); B cycles depth type when format is Depth. Each cycle re-creates the RenderTarget with the new format/type combo. The bridge's WEBGL_depth_texture surface advertises both names (WEBGL_depth_texture + OES_depth_texture) via getSupportedExtensions; getExtension for either returns an object exposing UNSIGNED_INT_24_8_WEBGL = 0x84FA. The post-pass shader runs end-to-end on native GLES via #pragma raw_passthrough — Three.js's #include <packing> directive expands into the perspectiveDepthToViewZ + viewZToOrthographicDepth helpers used to linearize the depth sample.
